# ユーザーガイド

open-maneki の使い方を説明します。インストールと起動の手順は
[README の Development 節](../README.md#development)を参照してください。

> このガイドは UI と同じ日本語で書かれています。英語版は UI の多言語対応時に
> 用意する予定です。
> This guide is written in Japanese to match the app's UI language.

## このアプリの性格

- **シングルユーザー・ローカル動作**。データは手元の SQLite ファイル
  （`prisma/dev.db`）に保存され、外部にはどこにも送信されません
- **市場価格は手入力**です（外部 API からの自動取得は将来対応予定）
- 対象資産は貯金・日本株・米国株・投資信託。口座は課税口座・NISA 口座・
  確定拠出年金を区別できます

## 初期設定の流れ

登録には順序があります。**口座 → 資産 → 取引 → 価格** の順に登録すると、
資産一覧に保有状況が表示されます。

### 1. 口座を登録する（「口座登録」ページ）

金融機関ごとに口座を作ります。

- **口座名**: 例「楽天証券」「○○銀行」
- **口座種別**: 課税口座 / NISA 口座 / 確定拠出年金

同じ金融機関の課税口座と NISA 口座は、同じ口座名で種別を変えて 2 つ
登録できます。

### 2. 資産を登録する（「資産登録」ページ）

**最初に現金資産を作るのがおすすめです**（例: 資産名「JPY 現金」・
資産種別「現金」・通貨 JPY）。現金への入出金や現金残高の表示は、
この現金資産に対して行われます。

続いて保有する銘柄を登録します。

- **資産種別と通貨**: 日本株は JPY、米国株は USD に固定されます。
  現金と投資信託は通貨を選べます
- **証券コード・ティッカー**: 例「7203」「AAPL」。任意です（現金には
  入力できません）
- **株主優待メモ**: 日本株のみ入力できます（任意）

### 3. 取引を登録する（「取引登録」ページ）

最初に現金資産への**入金**を登録すると、現金残高が表示されるようになります。
続いて銘柄の売買を登録します。

- **買付・売却**: 数量（株数・口数の整数）と金額を入力します。
  金額は**受渡金額（手数料込みの総額）**です — 単価ではありません
- **配当**: 配当実績の記録です。数量は入力しません
- **入金・出金**: 現金資産に対してのみ登録できます
- 金額の書式: JPY は円の整数、USD はドルで小数 2 桁まで（例: 123.45）

台帳の整合性が崩れる登録は自動的に拒否されます（保有数を超える売却、
残高を超える出金、過去の日付でさかのぼって矛盾が生じる登録など）。

### 4. 価格を登録する（「価格登録」ページ）

銘柄ごとに基準日と価格を登録すると、資産一覧に評価額が表示されます。

- **投資信託は 1 万口あたりの基準価額**を入力します（証券会社の表示と同じ）
- 同じ銘柄・同じ基準日で再登録すると**上書き**されます。入力ミスは
  同じ日付で登録し直せば訂正できます

### 5. 資産一覧で確認する（「資産一覧」ページ）

3 つの表が表示されます。

- **保有資産**: 口座 × 銘柄ごとの数量・取得原価・評価額
- **現金残高**: 口座 × 通貨ごとの残高
- **通貨別合計**: JPY と USD それぞれの合計（円換算はしません）

## 知っておきたい仕様

### 株を買っても現金は自動で減りません

このアプリの台帳は「1 件の取引 = 1 つの資産の記録」です（単式簿記）。
買付を登録しても現金資産の残高は変わりません。現金残高も正確に
合わせたい場合は、対応する**出金**（買付なら）や**入金**（売却・配当なら）を
自分で登録してください。現金を厳密に追わず保有状況だけ見る使い方も
できます。

### 全量売却した銘柄は一覧に表示されません

数量がゼロになった銘柄は保有資産の表から消えます。取引履歴は
「取引登録」ページの一覧に残ります。

### 価格未登録の銘柄は評価額が出ません

保有資産の表には「価格未登録」と表示され、通貨別合計からは除外されます
（除外した旨の注記が付きます）。「価格登録」ページで価格を入れると
表示されます。

## データのバックアップ

データはすべて `prisma/dev.db`（SQLite ファイル 1 つ）にあります。
このファイルをコピーすればバックアップになります。
